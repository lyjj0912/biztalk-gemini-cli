# [PRD] 업무 말투 변환기 (BizTalk Converter)

> **프로젝트 명**: BizTalk (업무 말투 변환기)
> **문서 상태**: Draft (v1.0)
> **실습 형태**: One Day 프로젝트 (약 4시간)
> **기본 원칙**: "완벽한 서비스보다 작동하는 서비스"를 우선으로 구현합니다.

---

## 1. 제품 개요 (Product Overview)
본 서비스는 사용자가 입력한 일상적인 말투의 업무 내용을 수신 대상(상사, 동료, 고객 등)에 맞춰 적절한 비즈니스 커뮤니케이션 어투로 자동 변환해주는 AI 기반 웹 서비스입니다.

### 1.1 배경 및 목적
- 업무 메시지 작성 시 표현의 적절성을 고민하는 시간을 단축합니다.
- 비즈니스 에티켓에 익숙하지 않은 신입 사원이나 실무자의 소통을 지원합니다.
- 핵심 내용은 유지하되, 상대방에 맞는 최적의 톤앤매너를 제공합니다.

---

## 2. 사용자 대상 (Target Audience)
- **신입 사원**: 상사나 외부 고객에게 보낼 메시지 작성이 어려운 사용자
- **실무 담당자**: 반복적인 업무 요청을 정중하고 빠르게 작성하고 싶은 사용자
- **비즈니스 커뮤니케이션 입문자**: 핵심 의도는 명확하나 격식 있는 문장 구성에 어려움을 겪는 사용자

---

## 3. 핵심 기능 요구사항 (Functional Requirements)

### 3.1 텍스트 입력 및 수신 대상 선택
- [ ] 사용자는 변환하고자 하는 원문 내용을 자유롭게 텍스트 영역에 입력할 수 있어야 함.
- [ ] 총 4가지의 수신 대상을 선택할 수 있는 인터페이스를 제공함.
    - **상사/임원**: 격식 있고 공손한 경어체
    - **타팀 동료**: 정중하면서도 협조적인 어조
    - **고객/외부**: 친절하고 신뢰감 있는 서비스 어조
    - **팀 내 동료**: 간결하고 실무적인 어조

### 3.2 AI 말투 변환 처리
- [ ] 입력된 텍스트와 선택된 대상을 기반으로 AI(Solar-Pro2) 모델에 변환 요청을 보냄.
- [ ] 각 대상별로 최적화된 시스템 프롬프트를 적용하여 변환 품질을 확보함.

### 3.3 결과 출력 및 활용
- [ ] 변환된 결과를 화면에 실시간으로 표시함.
- [ ] 변환된 텍스트를 한 번의 클릭으로 클립보드에 복사할 수 있는 기능 제공.
- [ ] API 호출 중에는 사용자에게 로딩 상태를 시각적으로 표시함.

---

## 4. 기술 스택 (Technical Stack)

| 영역 | 선택 기술 | 비고 |
| :--- | :--- | :--- |
| **Frontend** | HTML5, CSS3, Vanilla JS | 프레임워크 없이 순수 웹 기술 사용 |
| **Backend** | Python, FastAPI | 가볍고 빠른 API 서버 구축 |
| **AI/LLM** | Upstage Solar-Pro2 | 한국어 비즈니스 대화에 특화된 모델 |
| **API Framework**| LangChain | LLM 호출 및 프롬프트 관리 |
| **Deployment** | Vercel (Frontend), GitHub | 정적 페이지 배포 및 버전 관리 |

---

## 5. 시스템 아키텍처 (System Architecture)

1.  **Frontend (UI)**: 사용자로부터 원문과 수신 대상을 입력받아 Backend API 호출
2.  **Backend (FastAPI)**: 요청을 수신하여 해당 대상에 맞는 프롬프트와 함께 LLM에 전달
3.  **AI Engine (Upstage)**: Solar-Pro2 모델이 텍스트를 변환하여 응답 생성
4.  **Flow**: `Input -> FastAPI -> Solar-Pro2 -> FastAPI -> Output(UI)`

---

## 6. 개발 단계 및 로드맵 (Roadmap)

### STEP 1: 환경 준비 (30분)
- 프로젝트 디렉토리 구조 생성 및 Git 연동
- Upstage API 키 발급 및 환경 변수(`.env`) 설정
- 필수 라이브러리 설치 (`fastapi`, `uvicorn`, `langchain-upstage` 등)

### STEP 2: 백엔드 API 구현 (90분)
- FastAPI를 활용한 `/api/convert` 엔드포인트 개발
- 수신 대상별 프롬프트 템플릿 설계
- LangChain을 통한 Solar-Pro2 모델 연동 테스트

### STEP 3: 프론트엔드 구현 (60분)
- HTML/CSS를 이용한 메인 레이아웃 및 버튼 디자인
- JavaScript를 이용한 API 통신 로직 및 결과 표시 기능 구현
- 클립보드 복사 및 로딩 처리 로직 추가

### STEP 4: 배포 및 검증 (30분)
- GitHub 레포지토리 푸시
- Vercel을 통한 프론트엔드 배포
- 실제 모바일/PC 환경에서 동작 확인 및 최종 테스트

---

## 7. 제외 범위 (Out of Scope)
- 사용자 인증 및 로그인 기능
- 변환된 메시지의 DB 저장 및 이력 관리 기능
- 고도화된 UI/UX 디자인 (MVP 기능에 집중)

## 8. 성공 체크리스트 (Success Criteria)
- [ ] 텍스트 입력 후 버튼 클릭 시 5초 이내에 변환 결과가 출력되는가?
- [ ] 선택한 대상(상사 vs 팀 동료)에 따라 말투의 차이가 명확히 나타나는가?
- [ ] 복사하기 버튼 클릭 시 실제로 클립보드에 내용이 저장되는가?
- [ ] 외부 URL(Vercel)을 통해 서비스에 접속 가능한가?
